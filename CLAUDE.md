# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Environment

All scripts run inside the local venv. Always activate it first:

```bash
source venv/Scripts/activate   # Windows bash
python <script.py>
```

Install dependencies as needed:
```bash
pip install pandas scikit-learn numpy scipy matplotlib seaborn
```

## Project Purpose

This is a numerical / data-science research repo. It has two independent tracks:

**Track 1 — Surface / Signal Analysis**
- `roughness.py` — generates a 2D Gaussian-smoothed random surface via FFT convolution, saves as PNG. Key parameter: `gsigma` (controls correlation length).
- `correlation_length.py` — takes a glob of image files (`python correlation_length.py "*.png"`), computes 2D PSD → auto-correlation via FFT, plots 1D cross-sections.
- `heatmap_with_nonuniform_distribution.py` — reusable `plot_2D_heatmap()` function that interpolates scattered `(x, y, value)` points onto a regular grid with `scipy.griddata`, then renders with `plt.imshow`. Also generates `test_data.csv` (144 pts) when run directly.

**Track 2 — Key Factor Analysis Pipeline**
Three scripts that form a sequential pipeline:

```
test_data.csv
     ↓  preprocess.py
test_data_processed.csv
     ↓  model_train.py
console output (Lasso + RF feature importance)
```

- `preprocess.py` — reads a CSV, applies `StandardScaler` to specified float columns and `OneHotEncoder` to specified string columns. All configuration (input/output paths, column lists) is at the top of the file.
- `model_train.py` — reads the preprocessed CSV, performs stratified train/test split (Step 4), then trains Lasso (L1 LogisticRegression) and RandomForestClassifier (Step 5). Prints coefficients, feature importances, and a side-by-side summary. All configuration is at the top (`TARGET_COL` must be set to an existing column).

## Key Conventions

- Every script has a `# ─── Configuration ───` block at the top — edit only that block to adapt to a new dataset.
- `roughness.py` is git-ignored (listed in `.gitignore`) — do not commit it.
- Image outputs (`*.png`, `*.jpg`, etc.) are git-ignored.
- `test_data.csv` is the canonical test dataset (144 rows, columns: `x`, `y`, `value`); generated by running `heatmap_with_nonuniform_distribution.py`.
